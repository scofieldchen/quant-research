# STH-MVRV 报告生成技能

## 角色定义
作为资深金融分析专家，负责将 sth-mvrv 原始数据与图表转化为结构化的专业分析报告。

## 核心流程
请复制并执行以下任务清单，确保每一步都已完成：

```markdown
报告任务进度：
- [ ] 1. 读取数据：加载 `notebooks/sth_mvrv/outputs/summary.md`
- [ ] 2. 视觉分析：读取 `notebooks/sth_mvrv/outputs/` 下的所有 .png 图片并提取关键趋势
- [ ] 3. 撰写报告：基于数据与视觉分析结果填充报告模板
- [ ] 4. 存储结果：将最终报告保存至 `notebooks/sth_mvrv/report.md`
```

## 执行细节

### 1. 输入路径与要求
- **摘要数据**: `notebooks/sth_mvrv/outputs/summary.md`。必须提取最新的 sth-mvrv 数值和标准分数。
- **图表分析**: 遍历 `notebooks/sth_mvrv/outputs/` 目录。
    - 重点观察标准分数是否突破 0 线（多空分界）。
    - 观察标准分数是否超过 2.0（超买/高位风险）。
    - 观察 sth-mvrv 是否在 1.0 附近（盈亏平衡点）。

### 2. 报告模板
生成的报告必须严格遵循以下 Markdown 结构：

```markdown
## 数据解读

数据更新：{插入日期}

{描述核心数据及其短期变化。重点突出当前市场状态（上涨/下跌/盘整）和风险提示（如超买、趋势反转信号）}

## 回溯检验

{插入动量交易系统的回溯检验结果，包括胜率、盈利因子等关键指标}
```

### 3. 约束条件
- **严禁编造**: 所有数值必须来源于 `summary.md` 或图表观察。
- **路径规范**: 始终使用正斜杠 `/` 处理路径。
- **保存指令**: 撰写完成后，必须使用文件写入工具更新 `notebooks/sth_mvrv/report.md`。

## 示例报告

以下是符合要求的报告示例：

```markdown
## 数据解读

数据更新：2026-01-14

sth-mvrv（短期持有者市场价值和实现价值比例）当前为 0.99，极度接近 1.0 的盈亏平衡点。

sth-mvrv 标准分数已上升至 2.86，显示市场结构处于强势上涨状态。自 2025 年 12 月 25 日标准分数上破零线以来，中期趋势已由跌转多。

**风险提示**：当前标准分数高于 2.0，表明短期市场可能存在过度拉升，需警惕高位盘整或回调风险。只要标准分数保持在 0 以上，中期看涨逻辑依然成立。

## 回溯检验

基于 sth-mvrv 标准分数的中期动量交易系统（策略：>0 做多，<0 做空）表现如下：

- **时间范围**: 2024年至今
- **交易表现**: 总交易 50 笔，胜率 52%，盈利因子 5.12。
- **平均收益**: 每笔交易平均盈利 3.5%。

*注：回溯检验基于历史数据模拟，不代表未来真实表现。*
```
